# UI·창 동작

- **상태**: 해결 (개별 항목)
- **관련 코드**: `src/main.rs`, `src/editor_window.rs`, `src/tray_manager.rs`, `src/config_store.rs`

개발 초기부터 누적된 UI·창 관련 고민입니다. 별도 plan 파일은 없고 대화·구현 과정에서 해결했습니다.

## 해결한 항목

### 캐럿이 좌상단에 고정

- **원인**: 텍스트 전체 교체 후 선택 위치를 복원하지 않음.
- **해결**: `EM_GETSEL` / `EM_SETSEL`, UTF-16 길이 기준 위치 보정.

### 세로 스크롤바 이중 표시

- **원인**: 메인 창과 `EDIT` 컨트롤 모두에 스크롤 스타일.
- **해결**: 메인 창 `WS_VSCROLL` 제거, 에디터만 스크롤.

### 작업 표시줄에 창이 보임

- **요구**: 트레이 상주형이므로 작업 표시줄에는 나타나지 않게.
- **해결**: 보이지 않는 owner 창 + 일반 캡션 유지 (`WS_EX_TOOLWINDOW` 대신).

### 창 열기/닫기·위치 저장

- **요구**: 최초 실행 시 창 표시, 종료 시 위치·크기 저장, 다시 열 때 복원.
- **해결**: `config.json`의 `x`, `y`, `width`, `height`, 다중 모니터 이탈 시 `CW_USEDEFAULT` fallback.

### 트레이 클릭 동작

- **고민**: 한 번 클릭으로 토글할지, 포커스 기준으로 숨길지.
- **해결**: 싱글클릭 무동작, 더블클릭으로 표시/포커스. 닫기·최소화는 숨김 처리.

### 상태바 컨트롤이 안 보임

- **원인**: 상태바 `STATIC`이 형제 컨트롤 영역을 덮음.
- **해결**: 생성 순서 조정, `HWND_TOP`, `WS_CLIPSIBLINGS`.

### 타이틀바·여백

- **고민**: `WS_EX_TOOLWINDOW` 사용 시 닫기 버튼 등 캡션 레이아웃이 어색함.
- **해결**: owner 창 패턴 + 에디터 외곽 8px 마진.
